# Quick Start (Local)

This guide walks you through running your first TorcPy workflow on your local machine.
Everything — server, database, and job execution — runs on the same host.

## 1. Start the Server

```console
torcpy server run --db my_workflow.db
```

The server starts on `http://localhost:8080` by default. Keep this terminal open.

## 2. Write a Workflow Spec

Save this as `hello.yaml`:

```yaml title="hello.yaml"
name: hello-world

jobs:
  - name: greet
    command: echo "Hello from TorcPy!"

  - name: farewell
    command: echo "Goodbye from TorcPy!"
    depends_on: [greet]
```

## 3. Run the Workflow

In a second terminal:

```console
torcpy run hello.yaml
```

Output:

```
Created workflow 1
Initialized: 1 ready, 1 blocked
Running job 1: greet
Running job 2: farewell

Workflow 1 finished:
  Completed: 2
  Failed:    0
```

## Example: File-Dependency Pipeline

TorcPy can infer dependencies from shared files — no explicit `depends_on` needed:

```yaml title="pipeline.yaml"
name: file-pipeline

files:
  - name: raw
    path: data/input.csv
    st_mtime: 1710000000.0   # (1) marks this as a pre-existing input file
  - name: processed
    path: data/processed.csv
  - name: report
    path: output/report.txt

jobs:
  - name: preprocess
    command: python preprocess.py
    input_files: [raw]
    output_files: [processed]

  - name: analyze
    command: python analyze.py
    input_files: [processed]   # (2) TorcPy infers: analyze depends on preprocess
    output_files: [report]

  - name: notify
    command: echo "Pipeline complete!"
    depends_on: [analyze]
```

1. Setting `st_mtime` marks a file as pre-existing (not generated by the workflow). This
   distinguishes inputs from outputs.
2. Because `analyze` reads `processed`, which is written by `preprocess`, TorcPy automatically
   inserts a dependency.

```console
torcpy run pipeline.yaml
```
